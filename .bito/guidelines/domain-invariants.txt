Critical invariants for rich-text-renderer.rb. Violations here are bugs, not style issues.

ZERO RUNTIME DEPENDENCIES
- The gem must have no runtime gem dependencies. Only Ruby stdlib is allowed at runtime.
- Any new `gem.add_dependency` (non-development) in rich_text_renderer.gemspec is a violation.

XSS PROTECTION MUST NOT BE BYPASSED
- TextRenderer#render must call CGI.escapeHTML on node['value'] BEFORE applying any mark renderers.
- The Marshal deep-clone must happen BEFORE any mutation of the node. Never remove or reorder these.
- Marks wrap already-escaped text — this is the intended layering.

SDK AGNOSTICISM — NEVER IMPORT CONTENTFUL SDK
- Do not use `require 'contentful'` or check `is_a?(Contentful::Asset)` or any Contentful SDK class.
- Asset type detection must use ancestor-chain duck typing: asset.class.ancestors.map(&:to_s).any? { |name| name.include?('Asset') }
- Raw Hash assets must be detected by checking for `fields.file` structure.

DEFAULT MAPPINGS COMPLETENESS
- Every node type added to DEFAULT_MAPPINGS must have a corresponding renderer class.
- Every new renderer class must be required in the appropriate barrel file (block_renderers.rb, text_renderers.rb, etc.).
- The nil key in DEFAULT_MAPPINGS must always be present — it is the catch-all for unknown node types.

ENTRY BLOCK RENDERER IS A STUB BY DESIGN
- EntryBlockRenderer intentionally renders raw .inspect output. Do not add entry-field-specific logic here.
- Improvements to embedded entry rendering belong in consumer code, not this gem.

SIGNAL EXCEPTION STYLE
- Use `fail` not `raise` throughout the gem (enforced by RuboCop). Do not introduce `raise`.

RUBOCOP_TODO IS AUTO-GENERATED
- Never hand-edit .rubocop_todo.yml. Regenerate with: rubocop --auto-gen-config

ASSET RENDERING BEHAVIOR
- AssetBlockRenderer renders images (content_type includes 'image') as <img> tags; all other assets as <a> links.
- AssetBlockRenderer inherits from AssetHyperlinkRenderer — changes to asset hyperlink rendering affect both.
